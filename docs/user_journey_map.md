## 페르소나

**타깃 유저**: 개인 프로젝트 관리자 (프로젝트 오너 본인)

**특성**:
- 여러 TODO와 프로젝트를 동시에 관리 (2~3개 프로젝트, 최대 100개 미만 TODO)
- 바쁜 일상으로 인해 일정 관리 도구 사용이 불규칙적
- 업무 중압감 상황에서 앱 사용을 잊게 됨
- 기존 TODO 관리 도구(Notion, Trello)의 한계를 경험

**주요 Pain Points**:
1. 프로젝트 가시성 부족 - 칸반 보드에서 어떤 TODO가 어떤 프로젝트인지 파악 어려움
2. 일정 관리 앱 사용 중단 - 앱과 실제 상황 동기화 실패 → 사용 포기

## 여정 1: 바쁜 평일, TODO 관리 중단 → 재시작

### 배경
사용자가 업무가 바빠지거나 중압감 받는 상황에서 TODO 앱 사용을 잊고, 며칠 후 다시 접속하는 상황

### 여정 단계

| 단계 | 행동 | 생각/감정 | Pain Point | 해결 방안 (GTD-TODO) |
|------|------|-----------|-----------|---------------------|
| **1. 바쁜 업무 시작** | 급한 업무에 집중, TODO 앱 접속 중단 | "일단 이것부터 해야 해" | 앱 사용 잊음 | *(Phase 2: 리마인더/알림)* |
| **2. 며칠 후 앱 재방문** | 앱 열기 → 현실과 앱 상태 불일치 발견 | "아... 이거 다 업데이트해야 하나? 😰" | 동기화 작업 부담 | *(Phase 2: 페르소나 기반 위로 메시지)* |
| **3. 동기화 시도** | 완료된 TODO 체크, 새로운 TODO 추가 | "귀찮다... 나중에 하자" | 수동 동기화 번거로움 | Inbox에 빠르게 추가, 나중에 정리 가능 |
| **4. 앱 사용 포기** | 앱 닫기, TODO 관리 중단 | "역시 안 되네..." | 앱 사용 중단 | *(Phase 2: 지속 사용 유도 메커니즘)* |

### GTD-TODO의 개선점 (MVP)
- **빠른 Inbox 추가**: 일단 기록만 하고 나중에 정리 가능 (심리적 부담 감소)
- **GTD Reflect 단계**: 주기적으로 정리하는 워크플로우 제공
- **정리 강제하지 않음**: 정리 안 해도 앱 사용 가능

### GTD-TODO의 개선점 (Phase 2)
- 페르소나 기반 위로 메시지 ("괜찮아요, 바빴던 것 같네요")
- 자동 우선순위 재조정 제안
- 부드러운 재시작 유도

## 여정 2: 프로젝트 중심 작업 관리

### 배경
사용자가 여러 프로젝트를 진행하면서 특정 프로젝트에 집중하고 싶은 상황

### 여정 단계

| 단계 | 행동 | 생각/감정 | Pain Point | 해결 방안 (GTD-TODO) |
|------|------|-----------|-----------|---------------------|
| **1. 여러 프로젝트 진행 중** | 칸반 보드에서 TODO 확인 | "이 TODO가 어느 프로젝트였지?" | 프로젝트 가시성 부족 | **프로젝트 배지 표시** |
| **2. 특정 프로젝트 집중** | 해당 프로젝트 TODO만 보고 싶음 | "다른 건 나중에... 이것만 보자" | 관련 없는 TODO가 시야 방해 | **프로젝트 필터 기능** |
| **3. 프로젝트 진행 상황 파악** | 프로젝트별 완료 현황 확인 | "얼마나 남았지?" | 진행도 파악 어려움 | 프로젝트별 진행도 표시 |
| **4. 다음 할 일 선택** | Next Actions에서 선택 | "이제 뭐 하지?" | 우선순위 불명확 | GTD Next Actions 명확화 |

### GTD-TODO의 개선점 (MVP)
- **프로젝트 배지**: 각 TODO 카드에 프로젝트 이름 표시 (색상/라벨)
- **프로젝트 필터**: 특정 프로젝트 선택 시 해당 TODO만 표시
- **프로젝트별 진행도**: 완료율 시각화
- **GTD Next Actions**: 다음에 할 일을 명확하게 관리

## 여정 3: 일상적인 GTD 워크플로우

### 배경
사용자가 GTD 방법론에 따라 TODO를 관리하는 일상적인 프로세스

### GTD 5단계 워크플로우

| GTD 단계 | 행동 | 생각/감정 | 해결 방안 (GTD-TODO) |
|----------|------|-----------|---------------------|
| **1. Capture (수집)** | 아이디어/할 일이 떠오름 → Inbox에 추가 | "일단 적어두자" | 빠른 Inbox 입력 UI |
| **2. Clarify (명료화)** | Inbox 검토 → 실행 가능한지 판단 | "이건 할 수 있어? 언제?" | 실행 가능 여부 판단 지원 UI |
| **3. Organize (정리)** | Next Actions / Someday / Waiting For 분류 | "이건 다음에 할 일, 저건 나중에" | 드래그 앤 드롭 또는 빠른 이동 버튼 |
| **4. Reflect (검토)** | 주간 리뷰, 프로젝트 진행도 확인 | "이번 주는 뭘 했지?" | 리뷰 대시보드 *(Phase 2 고도화)* |
| **5. Engage (실행)** | Next Actions에서 선택 → In Progress → Done | "이제 이거 하자!" | 직관적인 칸반 드래그 앤 드롭 |

### GTD-TODO의 칸반 구조

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐     ┌──────┐
│   Inbox     │ ──> │ Next Actions │ ──> │ In Progress │ ──> │ Done │
└─────────────┘     └──────────────┘     └─────────────┘     └──────┘
                            │
                            ↓
                    ┌──────────────────┐
                    │  Someday/Maybe   │  (별도 영역)
                    └──────────────────┘
                    ┌──────────────────┐
                    │   Waiting For    │  (별도 영역)
                    └──────────────────┘
```

## 핵심 사용 시나리오

### 시나리오 1: 빠른 아이디어 캡처

**상황**: 회의 중 새로운 아이디어가 떠올랐을 때

**단계**:
1. 앱 열기 (빠른 실행)
2. Inbox에 "회의록 정리" 입력
3. 저장 후 앱 닫기
4. 나중에 Inbox 정리 시 프로젝트 할당 & Next Actions로 이동

**기대 효과**:
- 즉시 기록, 나중에 정리 (GTD Capture 단계)
- 생각이 떠올랐을 때 바로 기록하여 잊지 않음
- 정리는 나중에 할 수 있어 심리적 부담 없음

---

### 시나리오 2: 프로젝트 중심 작업

**상황**: "Go 백엔드 학습" 프로젝트에만 집중하고 싶을 때

**단계**:
1. 프로젝트 필터에서 "Go 백엔드 학습" 선택
2. 해당 프로젝트 TODO만 표시됨
3. Next Actions에서 "Go 투어 완료하기" 선택
4. In Progress로 이동 → 작업 시작
5. 완료 후 Done으로 이동

**기대 효과**:
- 집중력 향상 (다른 프로젝트 TODO가 시야에서 사라짐)
- 프로젝트 진행도 명확화
- 한 프로젝트에 몰입 가능

---

### 시나리오 3: 주간 리뷰 (GTD Reflect)

**상황**: 일요일 저녁, 한 주 정리 및 다음 주 계획

**단계**:
1. **Inbox 검토** → 각 항목을 분류
   - 실행 가능 → Next Actions
   - 언젠가 → Someday/Maybe
   - 대기 중 → Waiting For
2. **프로젝트별 진행도 확인**
   - 완료된 TODO 검토
   - 지연된 TODO 확인
3. **다음 주 우선순위 설정**
   - Next Actions 상단에 우선 TODO 배치

**기대 효과**:
- 정기적 정리 습관 형성
- 통제감 회복 ("내가 뭘 하고 있는지 안다")
- 다음 주 명확한 방향성 확보

---

### 시나리오 4: 타인 의존 작업 관리

**상황**: 동료에게 요청한 작업을 기다리는 중

**단계**:
1. "동료 A 코드 리뷰 기다림" TODO를 **Waiting For**로 이동
2. 주기적으로 Waiting For 영역 확인
3. 완료 통보 받으면 → Next Actions로 이동

**기대 효과**:
- 대기 중인 작업을 추적하여 놓치지 않음
- Next Actions가 깔끔하게 유지됨 (내가 바로 할 수 있는 것만)
- 다른 사람에게 의존하는 작업을 명확히 관리

---

### 시나리오 5: 장기 목표 관리

**상황**: "영어 공부"는 하고 싶지만 지금은 아닐 때

**단계**:
1. "영어 회화 학습" TODO를 **Someday/Maybe**로 이동
2. 여유 생길 때 Someday/Maybe 영역 검토
3. 시작할 준비 되면 → Next Actions로 이동

**기대 효과**:
- 머릿속 부담 덜기 ("지금은 안 해도 돼")
- 잊지 않고 보관 ("언젠가는 할 거야")
- Next Actions가 현실적으로 유지됨

## 사용자 경험 원칙

GTD-TODO는 다음 원칙을 바탕으로 설계됩니다:

1. **심리적 부담 최소화**
   - 정리를 강제하지 않음 (Inbox에 계속 쌓여도 OK)
   - 나중에 정리할 수 있는 유연성 제공

2. **프로젝트 가시성 극대화**
   - 프로젝트 배지로 한눈에 구분
   - 필터로 집중 모드 제공

3. **GTD 철학 준수**
   - 5단계 워크플로우 지원
   - Capture → Clarify → Organize → Reflect → Engage

4. **지속 가능한 사용**
   - 완벽함보다 지속성 우선
   - 앱 사용 중단 후에도 쉽게 재시작 가능 *(Phase 2 강화)*
