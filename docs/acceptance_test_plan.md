# 인수 테스트 작성법

## 인수 테스트 대상

인수 테스트는 사용자 관점에서 동작을 시스템에서 잘 수행하는 지 확인하는 테스트이다.  
인수 테스트는 어플리케이션 요소를 프로덕션에서 사용되는 것과 비슷한 환경에서 실행된다.  
인수 테스트는 사용자가 사용하는 기능 단위로 작성된다.  
인수 테스트는 기능에 필요한 모든 요소를 포괄한 테스트이므로, 비용이 크다. 그래서 성공 케이스 위주로 동작 검증에 한해 수행한다.  

## 인수 테스트 구조

인수 테스트는 테스트 배경 데이터 세팅, 사용자와 상호작용(API 호출), 응답 검증 등 인프라나 도메인 외부에 의존하는 복잡한 절차가 포함된다.  
그래서 인수 테스트를 작성할 때 이러한 복잡한 절차를 추상화하기 위해 아래 구조의 객체를 활용하여 인수 테스트를 작성한다.

- Senario : 사용자가 시스템을 통해 기능을 사용하는 흐름을 미리 정의된 DSL을 통해 정의한다.
- DSL : 사용자가 시스템과 상호작용하는 행위를 사용자 관점에서 서술한 도메인 특화 언어이다. (ex. 사용자가 로그인한다.)
- Driver : DSL로 정의된 사용자의 행위를 시스템 관점에서 구현한다. (ex. 로그인 API로 HTTP 요청을 전송한다.)
- SUT(System Under Test) : Driver가 행위를 구현하기 위해 사용하는 시스템 요소를 의미한다.
