## 정보 구조 (Information Architecture)

```text
GTD-TODO
│
├── 인증
│   ├── 로그인
│   ├── 회원가입
│   └── 로그아웃
│
├── 메인 대시보드 (칸반 보드)
│   │
│   ├── 헤더
│   │   ├── 로고
│   │   ├── 프로젝트 필터 드롭다운
│   │   └── 사용자 메뉴 (리뷰, 설정, 로그아웃)
│   │
│   ├── 칸반 영역 (메인 워크플로우)
│   │   ├── Inbox 컬럼
│   │   ├── Next Actions 컬럼
│   │   ├── In Progress 컬럼
│   │   └── Done 컬럼
│   │
│   └── 접기 가능 영역 (사이드 또는 하단)
│       ├── Someday/Maybe 섹션
│       └── Waiting For 섹션
│
├── 프로젝트 관리
│   ├── 프로젝트 목록
│   ├── 프로젝트 생성 모달
│   ├── 프로젝트 수정 모달
│   └── 프로젝트 삭제 확인
│
├── 리뷰 대시보드
│   ├── 주간 요약
│   │   ├── 완료된 TODO 목록
│   │   ├── 미완료 TODO 목록
│   │   └── 프로젝트별 진행도
│   │
│   └── Inbox 정리 모드
│       └── 빠른 분류 UI
│
└── 설정
    └── 사용자 프로필
```

## 주요 화면 구성

### 1. 메인 대시보드 (칸반 보드)

**핵심 화면**: 사용자가 가장 많은 시간을 보내는 곳

**레이아웃 구조**:

```text
┌─────────────────────────────────────────────────────────┐
│  [GTD-TODO 로고]    [프로젝트 필터 ▼]    [리뷰] [👤 ▼] │ ← 헤더
├─────────────────────────────────────────────────────────┤
│                                                           │
│  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐        │
│  │ Inbox  │  │  Next  │  │   In   │  │  Done  │        │
│  │        │  │Actions │  │Progress│  │        │        │
│  ├────────┤  ├────────┤  ├────────┤  ├────────┤        │
│  │ [+]    │  │        │  │        │  │        │        │
│  │        │  │        │  │        │  │        │        │
│  │ ┌────┐ │  │ ┌────┐ │  │ ┌────┐ │  │ ┌────┐ │        │
│  │ │TODO│ │  │ │TODO│ │  │ │TODO│ │  │ │TODO│ │        │
│  │ │Card│ │  │ │Card│ │  │ │Card│ │  │ │Card│ │        │
│  │ └────┘ │  │ └────┘ │  │ └────┘ │  │ └────┘ │        │
│  │        │  │        │  │        │  │        │        │
│  └────────┘  └────────┘  └────────┘  └────────┘        │
│                                                           │
├─────────────────────────────────────────────────────────┤
│  [▼ Someday/Maybe (3)]  [▼ Waiting For (2)]             │ ← 접기 가능
└─────────────────────────────────────────────────────────┘
```

**구성 요소**:

- **헤더**
  - 로고: 좌측 상단, 클릭 시 메인 대시보드 이동
  - 프로젝트 필터: 드롭다운 (전체/특정 프로젝트/프로젝트 없음/프로젝트 관리)
  - 리뷰 버튼: 리뷰 대시보드로 이동
  - 사용자 메뉴: 설정, 로그아웃

- **칸반 컬럼** (4개)
  - 각 컬럼: 제목, [+] 버튼, TODO 카드 목록
  - 드래그 앤 드롭 가능
  - 스크롤 가능

- **접기 가능 영역**
  - Someday/Maybe, Waiting For 섹션
  - 항목 수 배지 표시
  - 클릭 시 펼침/접힘 토글

### 2. TODO 카드 디자인

```text
┌─────────────────────────────────┐
│ [프로젝트 배지]           [···] │ ← 프로젝트 색상 배지 + 메뉴
│                                  │
│ TODO 제목이 여기 표시됩니다      │ ← 제목 (1-2줄)
│                                  │
│ 📝 간단한 설명 (선택)            │ ← 설명 (있으면 표시)
│                                  │
│ ☑️ 완료 표시                     │ ← 상태 (Done 컬럼에서만)
└─────────────────────────────────┘
```

**카드 구성 요소**:

1. **프로젝트 배지**: 프로젝트 이름 + 색상 (프로젝트 없으면 미표시)
2. **제목**: 1-2줄, 넘으면 말줄임표, 클릭 시 상세 모달
3. **설명**: 간단한 설명 (1줄, 선택)
4. **메뉴 버튼**: 수정, 삭제, 프로젝트 할당
5. **완료 체크박스**: Done 컬럼에서만 표시

## 인터랙션 플로우

### 플로우 1: 빠른 TODO 추가 (Inbox)

```text
[메인 화면]
  → [Inbox 컬럼의 [+] 버튼 클릭]
  → [간단 입력 폼 표시 (제목만)]
  → [Enter 또는 저장 버튼]
  → [TODO 카드가 Inbox에 추가됨]
  → [입력 폼 초기화 (연속 입력 가능)]
```

### 플로우 2: TODO 상태 이동 (드래그 앤 드롭)

```text
[TODO 카드 선택]
  → [드래그 시작]
  → [다른 컬럼으로 이동]
  → [드롭 가능 영역 하이라이트]
  → [드롭]
  → [카드 이동 + 상태 자동 업데이트]
  → [API 호출 (백그라운드)]
```

### 플로우 3: 프로젝트 필터링

```text
[헤더의 프로젝트 필터 클릭]
  → [드롭다운 메뉴 표시]
  → [프로젝트 선택]
  → [해당 프로젝트 TODO만 표시]
  → [필터 배지 표시 (현재 필터)]
  → ["전체 보기" 클릭 시 필터 해제]
```

### 플로우 4: 주간 리뷰

```text
[헤더의 "리뷰" 버튼 클릭]
  → [리뷰 대시보드 표시]
  → [주간 요약 확인]
  → [Inbox 정리 모드]
      → [각 TODO에 빠른 분류 버튼]
      → [버튼 클릭 → 해당 영역으로 이동]
  → [완료 TODO 확인]
  → [프로젝트 진행도 확인]
  → ["돌아가기" → 메인 대시보드]
```

## 디자인 원칙

### 1. 심플함 (Simplicity)

- 최소한의 UI 요소: 작업에 집중할 수 있도록 불필요한 장식 제거
- 명확한 시각적 위계: 중요한 요소가 눈에 잘 띄도록
- 여백 활용: 답답하지 않은 레이아웃

### 2. 직관성 (Intuitiveness)

- 드래그 앤 드롭: 상태 변경은 직관적인 드래그로
- 시각적 피드백: 모든 인터랙션에 즉각적인 피드백
- 명확한 라벨: 버튼, 메뉴 이름은 행동 중심 (예: "추가", "저장")

### 3. 접근성 (Accessibility)

- 키보드 내비게이션: 모든 기능을 키보드로 접근 가능
- 색상 의존도 낮춤: 색맹 사용자 고려, 텍스트 라벨 병행
- 적절한 대비: WCAG AA 이상 준수

### 4. 반응성 (Responsiveness)

- 모바일 대응: 테블릿, 모바일에서도 사용 가능
- 빠른 피드백: 로딩 없이 즉각적인 UI 업데이트 (Optimistic UI)
- 오프라인 고려: 네트워크 오류 시 명확한 안내

### 5. 부담 최소화 (Low Pressure)

- 정리 강제하지 않음: Inbox에 쌓여도 OK
- 긍정적 톤: 에러 메시지도 부드럽게
- 완료 강조: 완료한 것을 축하 (Done 컬럼)

## UI/UX 가이드라인

### 색상 팔레트

- **Primary**: 브랜드 색상 (선택 예정)
- **프로젝트 색상**: 파란색, 녹색, 빨간색, 노란색, 보라색 등
- **배경**: 밝은 회색 (#F5F5F5) 또는 흰색
- **텍스트**: 짙은 회색 (#333333)
- **경계선**: 연한 회색 (#E0E0E0)

### 타이포그래피

- **제목 (Heading)**: 16-20px, 볼드
- **본문 (Body)**: 14px, 레귤러
- **작은 텍스트**: 12px, 라이트
- **폰트**: 시스템 폰트 또는 Inter, Noto Sans

### 간격

- **컬럼 간격**: 16px
- **카드 간격**: 8px
- **내부 패딩**: 12-16px
- **모달 패딩**: 24px

### 인터랙션

- **호버 효과**: 배경 색상 약간 변화
- **드래그 중**: 카드 투명도 80%, 그림자 추가
- **드롭 가능 영역**: 배경 색상 변화 또는 점선 테두리
- **로딩**: 스피너 또는 스켈레톤 UI

## 반응형 디자인

### 데스크톱 (1024px 이상)

- 4개 컬럼 가로 배치
- Someday/Maybe, Waiting For는 하단 접기 가능 섹션

### 테블릿 (768px ~ 1023px)

- 2개 컬럼씩 2줄 배치
- 또는 스크롤 가능한 가로 레이아웃

### 모바일 (767px 이하)

- 1개 컬럼씩 세로 스크롤
- 또는 탭으로 컬럼 전환
- 드래그 앤 드롭 대신 버튼으로 상태 변경 가능
